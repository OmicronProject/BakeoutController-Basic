1 # User Interface {#user_interface}
2 
3 ## Introduction
4 The user interface consists of three components, a set of FXML files that
5 describe the UI markup, a set of Cascading Style Sheets (CSS) that describe
6 the UI styling, and a collection of Spring beans that work as controllers.
7 
8 ## FXML Markup
9 
10 [JavaFX](http://docs.oracle.com/javase/8/javase-clienttechnologies.htm)
11 provides XML-based markup for graphical elements as an alternative to
12 instantiating graphics in business logic. A consequence of this is that
13 concerns between UI markup and business logic remain well-separated. At boot
14  time, an [FXMLLoader](http://bit.ly/2jDLLPg) loads the ``Application.fxml``
15  file. This file is the entry point for the entire UI.
16 
17 Each tag represents a class. Each attribute in the tag represents a
18 constructor argument to the class, annotated with ``@NamedArg``.
19 
20 The ``<fx:include>`` tag is used to include other FXML files. The
21 ``<stylesheets>`` section is used to import CSS into the document.
22 
23 The ``<fx:controller>`` tag is used to indicate the controller that must be
24 instantiated in order to manage the particular element.
25 
26 The [FXMLLoader](http://bit.ly/2jDLLPg) is configured such that it uses
27 ``<ReturnType> ReturnType context.getBean(Class<ReturnType> requiredType)`` to
28  do this.
29 
30 Use the ``styleClass`` attribute to specify a string or an array of strings
31 representing the CSS classes to which the element belongs. Unlike in HTML,
32 CSS class names need to be separated by a comma and space, instead of
33 whitespace.
34 
35 ## CSS
36 
37 [Cascading Style Sheets (CSS)](http://bit.ly/1G0YUfU) work similarly to CSS
38 in HTML. Referencing elements by style class is done by preceding the class
39 name with a ``.`` and referencing elements by id is done by preceding the
40 name with a ``#``. However, JavaFX defines its formatting keys differently
41 than standard CSS. For more information, see the
42 [Oracle JavaFX CSS Guide](http://bit.ly/2jYG0PE).
43 
44 ## How to Make a UI Component
45 
46 1. Create an FXML file
47 2. Create a CSS file to style it
48 3. Create a controller and place a reference to the controller in the FXML file
49 4. Instantiate the controller in ``UserInterfaceConfiguration.java`` as a bean
